2.2. Grficos de barra realizando agregao de dados

Como a biblioteca seaborn tem o objetivo de simplificar nossa tarefa de criao de grficos, ela j possui funes que realizam a agregao de dados automaticamente. Isso significa que, ainda que passemos dados em que a unidade de anlise no a mesma que a desejada para o grfico (categoria sendo representada), a funo barplot() j realiza automaticamente a tarefa de agregar valores numricos.

No entanto, importante termos conscincia de como essa funo funciona, qual seu valor padro e como podemos alter-lo. Para isso, vamos criar um grfico de barras simples, passando nosso conjunto de dados e relacionando as variveis "mandato" e "valor".

In[]:
sns.barplot(
    y="mandato",
    x="valor",
    data=cartoes,
)
Out[]:
<Axes: xlabel='valor', ylabel='mandato'>

O que est acontecendo com esse grfico? Por que h uma linha no meio das barras criadas?

Como cada categoria de "mandato" aparece em mltiplas linhas, a funo barplot() est realizando a mdia dos valores da varivel "valor", para cada categoria. Esse o modo padro de funcionamento da funo.

Podemos conferir essas informaes na documentao da funo: https://seaborn.pydata.org/generated/seaborn.barplot.html. A primeira informao da documentao consiste justamente em uma linha de cdigo que nos mostra todas os argumentos (opes disponveis) para essa funo, cada um acompanhado de seu respectivo valor padro. Estamos preocupados, neste momento, com os argumentos estimator e errorbar.

O primeiro argumento (estimator) indica qual funo ser aplicada aos dados para realizar a agregao da varivel numrica. O valor padro desse argumento (estimator="mean") indica que ser calculada a mdia da varivel numrica para cada categoria. Ou seja, em nosso exemplo estamos calculando o valor mdio dos pagamentos realizados em cada mandato.

O segundo argumento que vamos analisar (errorbar) indica se queremos ou no adicionar barras de erro ao grfico e especifica tambm como essa barra ser criada. O valor padro desse argumento (errorbar=('ci', 95)) indica que a barra ir representar um intervalo de confiana de 95%.

No precisamos nos preocupar com o significado exato desse intervalo, mas importante saber que uma medida do erro do valor mdio est sendo estimada. Isso no se aplica ao nosso caso porque temos o valor de todas as transaes, no uma amostra.

Vamos remover isso.

In[]:
sns.barplot(
    y="mandato",
    x="valor",
    errorbar=None,
    data=cartoes,
)
Out[]:
<Axes: xlabel='valor', ylabel='mandato'>

A partir daqui vamos fazer todos os grficos com a opo de errorbar=None (importante que o N seja maisculo!).

Vamos agora incluir mais uma varivel categrica em nossa anlise. Vamos calcular o valor mdio dos pagamentos feitos em crdito e em dbito, em cada mandato. Para tanto, basta utilizar novamente o argumento hue, indicando que queremos cores diferentes de barras para cada categoria da coluna forma_pagamento.

In[]:
sns.barplot(
    y="mandato",
    x="valor",
    hue="forma_pagamento",
    errorbar=None,
    data=cartoes,
)
Out[]:
<Axes: xlabel='valor', ylabel='mandato'>

Agora, em vez de calcular o valor mdio vamos calcular o gasto total em cada mandato. Para isso, vamos mudar a funo de agregao. Se por padro a funo de agregao utilizada a mdia (estimator="mean"), queremos agora usar a soma (estimator="sum") para obter o valor total dos pagamentos realizados em cada mandato.

In[]:
sns.barplot(
    y="mandato",
    x="valor",
    estimator="sum",
    data=cartoes,
)
Out[]:
<Axes: xlabel='valor', ylabel='mandato'>

2.3. Controlando a cor dos grficos

Para ajustar a cor desejada dos grficos no seaborn, 2 argumentos podem ser utilizados: color e palette.

Quando queremos usar uma mesma cor para todo o grfico (quando no usamos a codificao de cor pelo argumento hue), indicamos a cor desejada com o argumento color.

Vamos usar 2 meios principais para escolher a cor que desejamos utilizar em nossos grficos. O primeira consiste em usar o nome da cor. Contudo, o seaborn no aceita qualquer nome de cor (ele no pode imaginar a cor, como ns faramos). Assim, h um conjunto pr-determinado e limitado de cores que podemos indicar pelo nome (assim como passamos o nome de colunas, usando aspas). Uma listagem completa dos nomes de cores aceitos pelo seaborn pode ser encontrada na documentao do matplotlib: https://matplotlib.org/stable/gallery/color/named_colors.html

Vamos destacar aqui a lista de cores mais longa, com um grande nmero de variaes:

E tambm um outro conjunto de cores que replicam a paleta de cores do programa Tableau:

Para usar essas cores, basta passar o nome da cor como uma string (ou seja, entre aspas) para o argumento color. Vamos ajustar nosso grfico por mandato usando esse argumento:

In[]:
sns.barplot(
    y="mandato",
    x="valor",
    color="tab:gray",
    estimator="mean",  # essa linha  opcional
    errorbar=None,
    data=cartoes,
)
Out[]:
<Axes: xlabel='valor', ylabel='mandato'>

Podemos tambm usar o cdigo hexadecimal da cor que desejamos. O cdigo hexadecimal uma forma de representar cores que comumente usada em design, e suportada tambm por linguagens de programao. Ele composto por 6 caracteres, que podem ser nmeros de 0 a 9 ou letras de A a F, e podemos escolher a cor que desejamos com preciso atravs de vrias ferramentas existentes.

Por exemplo, no prprio Google Search podemos pesquisar por "color picker" e escolher a cor que desejamos, ou acessar o link: https://g.co/kgs/hJsq1Hd. O cdigo hexadecimal da cor escolhida exibido na tela na caixa HEX.

Se quisermos usar essa forma de referenciar uma cor, em vez de passar o nome da cor basta passar o cdigo hexadecimal como uma string (entre aspas) para o argumento color. Vamos ajustar nosso grfico alterando a cor das barras para um tom especfico de azul.

In[]:
sns.barplot(
    y="mandato",
    x="valor",
    color="#134a91",
    estimator="mean",  # essa linha  opcional
    errorbar=None,
    data=cartoes,
)
Out[]:
<Axes: xlabel='valor', ylabel='mandato'>

Contudo, nem sempre queremos fazer grficos monocromticos! No geral, eles so uma boa ideia, mas podemos precisar de mais cores para:

  1. Representar categorias diferentes
  2. Destacar uma categoria especfica (e.g. a categoria que estamos destacando em nosso texto)
  3. Manter a coerncia das categorias do eixo X com outros grficos

Nesses casos, precisamos utilizar o argumento palette.

Vamos definir as cores que queremos usar para cada forma de pagamento. Para isso, vamos passar uma lista de cores para o argumento palette e reintroduzir o hue.

Para montar essa lista, devemos passar a sequncia de cores que desejamos usar separada por , entre colchetes ([ e ]). Podemos usar tanto os nomes das cores quanto os cdigos hexadecimais. Vamos ajustar nosso grfico para usar uma palheta de cores especfica.

In[]:
sns.barplot(
    y="mandato",
    x="valor",
    hue="forma_pagamento",
    palette=["tab:gray", "#134a91"],
    estimator="sum",
    errorbar=None,
    data=cartoes,
)
Out[]:
<Axes: xlabel='valor', ylabel='mandato'>
