Para isso, vamos utilizar a funo countplot() da biblioteca seaborn. Essa funo utilizada quando queremos construir um grfico de barras simples por meio da contagem de categorias em uma de nossas variveis.

1.2.2. Invertendo os eixos

Para reponder a essa pergunta, vamos criar um novo grfico, dessa vez utilizando outra varivel categrica presente nos nossos dados: o mandato presidencial de cada transao. No cdigo a seguir, comeamos novamente indicando os mandatos no eixo x:

O que voc acha desse grfico? Conseguimos ler bem o nome dos mandatos no eixo x? No, definitivamente no est legvel. Os nomes das categorias esto sobrepostos e torna-se muito difcil distinguir as categorias representadas no grfico.

Isso acontece porque a nossa linguagem se escreve da esquerda para a direita, ou seja, no mesmo sentido do eixo x de um grfico. Por conta disso, sempre que temos um grande nmero de categorias, ou nomes extensos das categorias, comum que surjam problemas de sobreposio dos nomes das categorias. Uma forma bem simples de lidar com isso (existem outras que veremos mais frente) mudar o eixo em que essas categorias esto representadas. 1.2.4. Usando cores para cruzar categorias

Vamos encerrar este primeiro captulo aprendendo a cruzar duas variveis categricas, que sero representadas como barras de cores diferentes em nosso grfico. Para isso, basta utilizar o argumento hue, que associada cada categoria a uma cor e cria uma legenda em nosso grfico para indicar as cores. Precisamos definir de qual coluna dos nossos dados vamos extrair as categorias que sero representadas como cores distintas no grfico.

Observe o cdigo a seguir. Nele estamos criando um grfico de contagem de transaes por mandato, colocando as categorias de mandato no eixo y (y="mandato"). Alm disso, estamos contando separadamente, em cada mandato, o nmero de transaes realizadas em cada forma de pagamento, crdito ou dbito (hue="forma_pagamento").

In[]:

    y="mandato",
    hue="forma_pagamento",
    data=cartoes
)
Out[]:
<Axes: xlabel='count', ylabel='mandato'>

O grfico nos permite observar que houve uma clara tendncia de reduo do nmero de transaes realizadas em dbito ao longo dos mandatos presidenciais. Essa tendncia, inclusive, parece se confirmar mesmo quando o nmero total de transaes cresce de um mandato para o outro. O grfico que criamos j capaz de evidenciar, portanto, uma relao complexa entre as informaes disponveis acerca dos pagamentos, algo que dificilmente conseguiramos identificar facilmente apenas trabalhando diretamente os dados tabulares, sem tentar produzir visualizaes.
